# 0001 - Uso do Expo Managed Workflow

Durante o desenvolvimento do aplicativo Habit+, optou-se pela utilização do framework Expo no modo Managed Workflow. Essa decisão foi tomada considerando a necessidade de um ambiente de desenvolvimento simples, ágil e com boa integração com ferramentas já conhecidas, como o Firebase.

O Expo Managed Workflow permite criar e testar aplicações móveis sem a necessidade de configurar manualmente o Android Studio ou o Xcode. Essa abordagem reduz significativamente o tempo de configuração e manutenção do ambiente, permitindo que o foco principal do desenvolvimento seja o código e as funcionalidades do aplicativo.

O Habit+ é um aplicativo leve, sem dependência de recursos nativos complexos, como Bluetooth, GPS ou sensores específicos. Dessa forma, o Expo atende plenamente às demandas do projeto, oferecendo suporte adequado para os recursos necessários, como autenticação, banco de dados e notificações push via Firebase.

Além disso, o Expo proporciona um fluxo de trabalho unificado entre Android e iOS, o que facilita o processo de testes e distribuição. A simplicidade do gerenciamento de dependências e a estabilidade das atualizações também foram fatores decisivos na escolha do framework.

Em comparação com alternativas como o React Native CLI e o Flutter, o Expo Managed Workflow apresentou a melhor relação entre praticidade, produtividade e compatibilidade com o ecossistema JavaScript, tornando-se a escolha mais adequada para os objetivos do Habit+.
