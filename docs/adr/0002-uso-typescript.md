# 0002 - Uso de TypeScript em vez de JavaScript

A decisão de utilizar TypeScript em vez de JavaScript puro no desenvolvimento do aplicativo Habit+ foi baseada na necessidade de um código mais seguro, padronizado e escalável. Por ser uma linguagem com tipagem estática, o TypeScript permite que erros sejam identificados ainda em tempo de compilação, evitando falhas comuns durante a execução e reduzindo a ocorrência de bugs relacionados a variáveis indefinidas ou incorretamente declaradas.

Como o Habit+ contará com diversas telas e componentes React interligados, o uso de tipagem explícita facilita a manutenção e a consistência entre as diferentes partes do sistema. A clareza proporcionada pelo TypeScript melhora a legibilidade do código e auxilia na colaboração entre desenvolvedores, garantindo um padrão unificado em toda a base do projeto.

Outro fator importante é a integração com o Firebase, utilizado para autenticação e armazenamento de dados. O TypeScript oferece excelente suporte para as bibliotecas do Firebase, fornecendo autocompletar, verificação de tipos e melhor detecção de erros em tempo de desenvolvimento. Essa compatibilidade contribui para um fluxo de trabalho mais eficiente e um código mais confiável.

Dessa forma, o uso do TypeScript representa uma escolha técnica que prioriza qualidade, segurança e organização do código, alinhando-se aos objetivos de robustez e escalabilidade do Habit+.
